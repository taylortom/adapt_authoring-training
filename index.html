Welcome

This page is the central meeting place for all things related to Kineo's fork of the Adapt authoring tool. See the links at the top of the page.

If you're not sure where you should be, the following may help:

Need information?
Have a look in the wiki attached to our adapt_authoring respository.
Got a bug to report, or a feature to request?
Head to the issues page on our adapt_authoring repository. Instructions for using it are in the wiki.
New to the tool?
We've set up a training exercise for newcomers which you can check out here.
Never heard of the Adapt project?
You're probably in the wrong place, the project's community page is a good place to start.

Why fork?

Don't worry, we're still friends.

Not every feature we develop at Kineo is suitable for the open-source project; be it unstable, or not within the scope of the open-source team's goals. These kinds of features obviously have no place in the open-source repo.

We also like to develop new open-source features in a separate repo (at least to begin with), as it keeps the open-source repo a bit more focussed and clutter free. We usually push the code to the open-source repo once it's ready for testing.

This being said, everything that we develop internally is stored in the repo, and is open-source and freely available to download via GitHub (and mostly importantly, it's all compatible with the open-source code).

Feedback and support

We welcome any feedback you may have regarding the authoring tool. We collate everything using GitHub's issues, including: bugs, feature requests and other general questions relating to functionality. If you have something to report, please log an issue (a GitHub account is required for this).

If you need help, the first thing to do is have a dig around in the GitHub wiki; your problem may already be documented there. If you can't find what you're looking for there, please get in touch with Tom Taylor or Pete Smith.
